finance-dl is a Python toolkit that automates downloading personal financial data from banks, credit card issuers, brokerages, and other financial services. It handles login flows, multi-factor authentication, and data extraction so you can keep your financial records up to date without manual effort. It is specifically designed to work alongside [beancount-import](https://github.com/jbms/beancount-import) for double-entry bookkeeping.

## What finance-dl Does

finance-dl connects to financial institutions through two mechanisms: direct OFX protocol connections (no browser required) and Selenium-based web scraping using ChromeDriver. Each scraper is configured with a simple Python function and writes downloaded data to a local directory.

## How It Works

<Steps>
  <Step title="Install the package">
    Install finance-dl from PyPI with `pip install finance-dl`. ChromeDriver is bundled via the `chromedriver-binary` package.
  </Step>
  <Step title="Write a config module">
    Create a Python file with `CONFIG_<name>()` functions — one per financial account. Each function returns a dict specifying the scraper module, credentials, and output directory.
  </Step>
  <Step title="Run a scraper">
    Execute `python -m finance_dl.cli --config-module my_config --config mybank` to download data for one account. Files are saved to your specified output directory.
  </Step>
  <Step title="Automate everything">
    Use `python -m finance_dl.update` to run all configurations in parallel, skip recently-updated ones, and log results to files for review.
  </Step>
</Steps>

<Note>
  finance-dl is designed to integrate with [beancount-import](https://github.com/jbms/beancount-import). The output files — OFX, CSV, JSON, PDF, and HTML — are formatted to be consumed by beancount-import importers for automated double-entry bookkeeping.
</Note>
